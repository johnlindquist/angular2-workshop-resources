01. Hello World

Hello World

TL;DR

You bootstrap an App Component to startup your app in the <app> tag of your HTML.

index.html, main.ts, app.ts

index.html

Your main index.html file will be required to simply include the compiled and bundled version of your code and declare a tag that defines the entry point of your app.

Something as empty and as simple as the following will work great:

<!DOCTYPE html>
<html lang="en">
<head>
  <title>Angular 2 StarWars Party!</title>
      <!--<base> is required for routing -->
  <base href="/">
</head>
<body>
<!-- <app> is the tag your Angular 2 app will be loaded into-->
<app>
  Loading...
</app>

<!-- Place your script tag here. Often your build will inject it for you.-->
</body>
</html>

app.ts

Your app.ts is your main application component.

The selector must match up with the <app> tag you used in your index.html. If the selector is party-time, the tag would need to be <party-time></party-time>.

The template is the main HTML of your project.

import {Component} from 'angular2/core';

@Component({
  selector: 'app',
  template:`
  Hello World!
`
})
export class App {}

The resulting body when run in the browser will look like the following. Notice how the Hello World is wrapped inside of an <app> tag even though your main template does not have any tags:

<body>

<app>
  Hello World!
</app>

<script src="app.bundle.js"></script>

</body>

main.ts

Import your App Component into your main.ts to bootstrap and startup your application. You can consider main.ts the entry point of your app and is only used for configuation and startup.

bootstrap returns a Promise, so you can use then and catch to handle bootstrap completion or errors.

import {bootstrap} from 'angular2/platform/browser';
import {App} from './app';

bootstrap(App)
  .then(()=> console.log(`App Running...`))
  .catch(err => console.log(err));

What's That Syntax?

Import and Export (JavaScript)

import and export work together to help you share code between files.

  • export lets you expose the code you want to share
  • import lets you pull in the exported code

Assuming both of the files below are in the same directory:

foo.js

export var message = "hi";

bar.js

import {message} from './foo';
console.log(message); //logs "hi"

Angular 2 Modules (JavaScript)

You will see many modules imported such as:

  • angular2/core
  • angular2/platform/browser
  • angular2/common

Just think of these as file paths to the code that you want to import.

Decorators (TypeScript)

@Component is a Decorator that changes the behavior of the class defined directly underneath it.

Decorator() is really just a function that takes the object with {selector:'app'} and applies it. The @ simply links that function to what's beneath it in the code. So when you see a Decorator think, "this is a function being invoked with what's declared next to it".

Classes (JavaScript)

The class keyword in JavaScript is just syntax sugar for what's been done in JavaScript for years.

class Person{}
var person = new Person();

is the same as

function Person(){}
var person = new Person();

It's simply much shorter and easier to write:

class Person {
    constructor(message) {
        this.greeting = message;
    }
    greet() {
        return "Hello, " + this.greeting;
    }
}

than

function Person(message) {
        this.greeting = message;
    }
    Person.prototype.greet = function () {
        return "Hello, " + this.greeting;
    };

Classes create the same prototype chains we know from previous versions of JavaScript, but with an arguably saner syntax.

()=> Arrow Functions

()=> is used just like an anonymous function(){}, but it doesn't create a this.

var sayHi = ()=> console.log('hi');

is equivalent to:

var sayHi = function(){console.log('hi')};

If there's only one argument, you can actually omit the ():

var sayHi = message => console.log(message);

is equivalent to 

var sayHi = function(message){console.log(message)};
