08. Loading Data with Http

Loading Data with Http

TL;DR

  • Inject HTTP_PROVIDERS into your app providers
  • Inject Http into your Service
  • Use http.get() to define the request
  • Use an async pipe | async to "subscribe" to the request

Http

The http service is included in the angular2/http package. You can actually use whatever http services you'd like with Angular 2, http is simply a convenience.

To use the http service, you'll need to include the HTTP_PROVIDERS either in the same Component's providers:[] or in a parent Component's providers:[]. Typically, you'd see the HTTP_PROVIDERS in you App Component:

@Component({
    selector: 'app',
    providers: [HTTP_PROVIDERS], //Http is now available to children components
    template:`
        hi
    `
})
class App{}

Then in any child component (a "smart" one!) you'd inject the Http service itself:

@Component({
    selector: `some-child-component`,
    template:`yo`
})
class SomeChildComponent{
    constructor(http:Http){
        //do http things
    }
}

http is built upon Observables. So when you call http.get(), it's not going to make the request until you subscribe to it.

//just sits there
http.get('http://localhost:4000')

//actually makes the request
http.get('http://localhost:4000').subscribe();

What this means is that you can define a request, assign it to a variable/field/whatever, then subscribe to that variable.

const example = http.get('http://localhost:4000');

//pass "example" around, then subscribe
example.subscribe();

The Async Pipe | async

You can subscribe in a template with the | async. This means that you can make the assignment, pass it into the template, then render out the results in the template whenenver the results are returned.

@Component({
    selector:'example',
    template:`        
        {{people | async}}
    `
})
class Example{
    people;
    constructor(http:Http){
        this.people = http.get('http://localhost:4000/people');
    }
}

mapping the Response to JSON

Many services requests will return JSON objects, so any easy way to get the JSON object is to map the response.

const example = http.get('http://localhost:4000')
    .map(res => res.json())

res.json() will grab the body as JSON and return it so you'll have convenient access. map is an "operator" in RxJS, so to use it, you'll need to import the map operator separately.

import 'rxjs/add/operator/map';

Gotchas

  • Remember to import rxjs/add/operator/map when using get().map

Moving Http to a Service

To be able to share your various requests across your app, you can move Http into its own Service:

@Injectable()
class ExampleService{
    people;
    constructor(http:Http){
        this.people = http.get('http://localhost:4000')
            .map(res => res.json())
    }
}

This way you can inject the Service into your Component (and any other Component for that matter):

@Component({
    selector: 'example-component',
    providers: [ExampleService],
    template: `
        {{exampleService.people | async}}
    `
})
class ExampleComponent{
    constructor(public exampleService:ExampleService){}
}

Show Me the Code

src/services/starwars.service.ts

import {Injectable} from 'angular2/core';
import {Http} from 'angular2/http';
import 'rxjs/add/operator/map';

const STARWARS_API = 'http://localhost:5000';

@Injectable()
export class StarWarsService{
    people;
    starships;
    vehicles;

    constructor(http:Http){
        this.people = http.get(`${STARWARS_API}/people`)
            .map(res => res.json());

        this.starships = http.get(`${STARWARS_API}/starships`)
            .map(res => res.json());

        this.vehicles = http.get(`${STARWARS_API}/vehicles`)
            .map(res => res.json());
    }
}

src/app.ts

import {Component} from 'angular2/core';

import {StarWarsService} from './services/starwars.service';
import {HTTP_PROVIDERS} from 'angular2/http';
import {Home} from './home/home.component';

@Component({
    selector: 'app',
    providers: [StarWarsService, HTTP_PROVIDERS],
    directives:[Home],
    template: `
        <home></home>
    `
})
export class App{
}

src/home/home.component.ts

import {Component} from 'angular2/core';
import {StarWarsService} from '../services/starwars.service';

@Component({
    selector: 'home',
    template: `
        <h2>People</h2>
        <div *ngFor="let person of starwarsService.people | async">
        {{person.name}}
      </div>

      <h2>Starships</h2>
        <div *ngFor="let starship of starwarsService.starships | async">
        {{starship.name}}
      </div>

      <h2>Vehicles</h2>
        <div *ngFor="let vehicle of starwarsService.vehicles | async">
        {{vehicle.name}}
      </div>
    `
})
export class Home{
    constructor(public starwarsService:StarWarsService){}
}

@Injectable()

The @Injectable() decorator is required so that Angular 2 has the proper Metadata to be able to properly register your service with the Injector. You don't need to worry about the metadata, it's handled during compilation from TypeScript to JavaScript. (This is also why "emitDecoratorMetadata": true, is required in your tsconfig.json when compiling down your TypeScript.)

Importing vs. Injecting

It's easy to get confused between import and Injecting, but they're not at all the same thing.

Importing (using the import keyword) is simply how JavaScript shares code across files.

Injecting happens during "runtime" (when your application is running in the browser) and is how you share values/instances across your application.

What's That Syntax?

Async Pipe | async

While many pipes are used for formatting data, the Async Pipe is used to allow data to be pushed into the template through either an Observable or a Promise.

The simplest example being a timer:

import {Component} from 'angular2/core';
import {Observable} from 'rxjs/Observable';
import 'rxjs/add/observable/interval';
@Component({
    selector: 'home',
    template: `
        {{timer | async}}
    `
})
export class Home{
    timer = Observable.interval(1000)
}
