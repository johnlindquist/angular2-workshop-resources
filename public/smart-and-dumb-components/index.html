04. Smart and Dumb Components

Smart and Dumb Components

TL;DR

Smart Components

  • Specific to your App
  • Are Containers for Dumb Components
  • Handle passing Data around (covered later)
  • Handle routing (covered later)
  • Use providers:[]

Dumb Components

  • Are reusable across apps
  • Use @Input() to receive data from the container
  • @Input() foo; is handled by [foo]="bar" in the container
  • Use @Output() to push data out to the container
  • @Output() is a new EventEmitter which emit() data out
  • @Output() foo = new EventEmitter(); is triggered with foo.emit() then handled by (foo)="onFoo()" in the container

Smart Components

There's only one Component, but some Components are specific to your application while others are going to be reusable.

We'll call a Component that's specific to your application a "smart" component. This means that your routes, data, and application wiring would all hook together using these types of Component. In practice, this means that "smart" Component use providers:[] for hooking up your application into the Component.

These Components also load in "dumb" Components, so their templates will typically be larger and have a few "dumb" components listed in their directives array.

Dumb Components

A reusable Component is a "dumb" Component. This Component shouldn't know anything about your application. No routes. No services. Nothing application specific.

All data in dumb Components should be passed in through @Input(). All data going out should be pushed out with @Output(). Using providers:[] in "dumb" components is rare and never used for application data.

Show Me the Code

src/smart-component.ts

import {Component} from 'angular2/core';
import {Dumb} from './components/dumb.component';

@Component({
    selector: 'smart',
    directives: [Dumb],
    template: `
    <dumb [message]="memo" (open)="onOpen($event)"></dumb>    
`
})
export class Smart{
    onOpen(event){
        console.log(event);
    }

    memo = "You've got mail";
}

src/components/dumb.component.ts

import {Component, Output, Input, EventEmitter} from 'angular2/core';

@Component({
    selector: 'dumb',
    template: `
    <button (click)="onClick()"></button>

`
})
export class Dumb{
    @Input() message;
    @Output() open = new EventEmitter();

    onClick(){
        this.open.emit('message opened!');
    }
}
