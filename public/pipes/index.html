09. Pipes

Pipes

TL;DR

  • Decorate a class with @Pipe
  • name your pipe
  • Add a transform method to your pipe
  • transform is invoked with (input, arg1, arg2, etc)
  • Return whatever output you'd like from the pipe
  • Include the pipe in your Component pipes:[]
  • Use your pipe in the template {{data | myPipe:arg1:arg2}}

Pipes

If you're coming from Angular 1, you'll soon notice the lack of a filter pipe and others that you may have come to rely on. Fortunately, pipes are extremely easy to create and share. Personally, I expect a large library of reusable Angular 2 pipes to be coming in the future.

| json Pipe

The | json pipe is extremely helpful for debugging in templates.

{{myObject | json}}

Built-in Formatting Pipes

| date, | uppercase, | lowercase, | currency, | percent are all available to format your template expressions:

{{ birthday | date:"MM/dd/yy" }}

Chaining Pipes

Pipes can be used one after the other. The output of the previous pipe will be used as the input for the next:

<!-- Results in something like "MONDAY, MAY 2, 2016 -->
{{ birthday | date | uppercase}}

Custom Pipes

You create a custom pipe using a @Pipe() decorator on a class. The @Pipe() requires a name, then you simply create a transform method that will take your template variable as input, then output whatever you return. Often, you're either formatting a string or manipulating an array.

Show Me the Code

import {Pipe} from 'angular2/core';
@Pipe({
  name: 'search'
})
export class Search{
  transform(data, key, term = ""){
    if(!data) return null;
    return data.filter(item => {
      return item[key].toLowerCase().includes(term.toLowerCase());
    })
  }
}

@Component({
  selector: 'person-list',
  directives: [Card],
  //include your pipe
  pipes:[Search],
  template: `
<input [(ngModel)]="name" type="text">

<div class="card-container">
  <card
    <!-- use your pipe -->
    *ngFor="#person of people | search:'name':name"
    [person]="person">
  </card>
</div>
`
})
export class PersonList{}
