10. Routing

Routing

TL;DR

  • Routing is very configuration heavy, be patient
  • Configure routes for your components by decorating with @RouteConfig
  • Define routes like {path: '/party', name: 'Party', component: Party}
  • The router loads a component into the <router-outlet>
  • Route to a name with [routerLink]="['Party']"
  • You build routes in a confusing array syntax: [routerLink]="['/ChildItem', {id: 'someId'}, 'SubItem1', {itemName: 'someName'}]

<base href="/">

Don't forget to include this in the <head> of your index.html, or else routing won't know where to start:

  <base href="/">

@RouteConfig()

With @RouteConfig() your App Component basically becomes a target for where the router will render components.

So an App component will typically have:

  • Navigation
  • <router-outlet> (where your components are loaded)
  • Header/Footer/etc

Decorate your App component with your root routes so that when your application loads, it knows which components to load. Loading a Home component would look like:

  @Component({
    //selector, template, etc
  })
  @RouteConfig([
    {path: '/home', name: 'Home', component: Home, useAsDefault: true}   
  ])
  export class App{}

Gotcha

Notice how the App is decorated with both @Component AND @RouteConfig. The nature of decorators allows you to stack them one on top of the other as long as the code immediately following is something that can be decorated, e.g. a class.

<router-outlet>

Now you need somewhere to render out your Home component.

Compare using the Home component as a directive:

  @Component({
        //selector, etc
        directives: [Home],
        template:`
            <home></home>
        `
  })
  export class App{}

Vs. loading the Home component as a route:

  @Component({
    //selector, etc
        template:`
            <router-outlet></router-outlet>
        `
  })
  @RouteConfig([
    {path: '/home', name: 'Home', component: Home, useAsDefault: true}   
  ])
  export class App{}

[routerLink]

Using [routerLink] directive allows you to hook into the routing system from your template. To navigate to your Home component, you add a [routerLink] that looks like the following:

<a [routerLink]="['Home']">Home</a>

.router-link-active

.router-link-active is a style that is automatically applied added to an element with [routerLink] defined when that link is active. This is great for highlighting in navigation.

And Beyond...

Going in-depth into router is outside the scope of this workshop, but the general gist through code samples:

  @RouteConfig([
    //"..." allows "children"
    {path: '/person/...', name: 'Person', component: Person}   
  ])

//The Person component would *also* have a `@RouteConfig()` to handle its children 
@RouteConfig([    
    // the "/" signifies a "terminal" route. These are required so the route 
    //at least stops somewhere
    {path: '/detail/', name: 'Detail', component: Detail}   
  ])
export class Person{}

A param in a route:

 { path: '/person/:id/', component: Person, name: 'Person' },

Matches up with a param in a constructor:

constructor(params: RouteParams) {    
    this.id = params.get('id');
}

The router has "lifecycle hook" decorators to allow or prevent navigation:

@CanActivate, @OnActivate, @CanDeactivate

//Takes a function invoked with the "next" route and the "previous" route.
//Return a boolean or a Promise that resolves to a boolean.
@CanActivate((next, prev) => boolean | Promise<boolean>)
