02. First Component

First Component

TL;DR

You make a component with a @Component decorator and a class. The @Component requires a selector and a template, then you include the component in a parent Component using directives: [].

Components

Components are the backbone of Angular 2. You will be writing a lot* of components, so you'll get accustomed to the new syntax quickly .

A Component is the combination of a class and a @Component decorator.

class

The class is a typical JavaScript class whose purpose is to support the template of the component. For example, you'll handle events from the template inside your class.

@Component

The @Component is a Decorator that maps the selector and the template to the class.

selector

Every @Component needs a selector so that it knows where it is declared in parent templates.

template

Every @Component also needs a template that will be rendered wherever the component is included.

Ambiguity Alert!

"Component" is used interchangably to describe the @Component decorator as well as the combination of the decorator and the class.

The class is often called a controller. That's mostly a due to the naming conventions of Angular 1, but the name suits it fine.

A @Component is a @Directive with a template, so you include your component class in the parents' directives:[] array.

I'm so confused...

Relax . All components look alike. In fact, the patterns will become very easy to read when you see many of them together. After you've created your third or fourth component, you'll be used to the patterns (you might even be copy and pasting from others components) enough that you won't really care about the syntax anymore.

Show Me the Code

src/components/first.component.ts

import {Component} from 'angular2/core';

@Component({
    selector: 'first',
    template: `
            I have a message for you: 
        `
})
export class First{
    message = "Hi!";
}

src/app.ts

import {Component} from 'angular2/core';
import {First} from './components/first.component';

@Component({
    selector: 'app',
    directives:[First],
    template: `
        <first></first>
    `
})
export class App{
}

Gotchas

If you have a blank page, check these first:

  • You included your component in the directives:[] Array
  • Your selector:'' name matches the <selector> in the template

What's That Syntax?

`` Multiline Strings

The backticks are used for multiline strings which makes them perfect for HMTL templates:

var s = `I am a string`;

var template = `
    <div>I am also a string!</div>
`;
