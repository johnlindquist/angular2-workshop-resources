03. Events and Refs

Events and Refs

TL;DR

  • Use (click)="onClick($event)" to handle events in templates
  • Use (keyup.enter)="onEnter($event) to handle key events
  • Use <input #myInput> to reference elements in the template

Handling Events

Events are part of every application. Your user is going to click, type, resize, etc, etc, so you'd better know how to accomodate them !

Events are handled by wrapping the event name inside of (). So if you want to handle a click on a button, you type:

<button (click)>

You add a "handler" for the event by assigning it a handler you've defined in your component:

In your template

<button (click)="onClick()">Click Me!</button>

In your Component

onClick(){
    console.log('clicked!');
}

The $event Object

To pass along the actual event, use $event in the handler

In your template

<button (click)="onClick($event)">Click Me!</button>

In your Component

onClick(event){
    console.log(event);
}

Keyboard Events

Want to know when a specific key was pressed? Check this out!

  <input
   (keyup.enter)="onEnter($event)"
   (keyup.esc)="onEscape($event)"
   (keyup.control.a)="onControlA($event)"
   (keyup.control.shift.l)="onControlShiftL($event)"
   >

Add the keyboard combo using the syntax above will allow you filter out to only the specific keyboard event you want!:tada:

Refs

You can reference values from elements in your template with a ref. You create a ref using # and a name on any element:

<input #example>

Elsewhere in the same template

{{example.value}}

Events Required for Changes!

The above example will not automatically update! You need to have some sort of event firing for it to catch the update. Something as simple as adding a (keyup) will do the trick:

<input #example (keyup)="noop">

Obviously, the above could be handled better with [(ngModel)] (discussed later), but it's definitely good to know!

Passing #ref Values to Event Handlers

You can also grab that value off of the input and pass it along to a (click) handler:

<input #example>
<button (click)="onClick(example.value)">Click Me!</button>

Show Me the Code

src/components/example.component.ts

import {Component} from 'angular2/core';

@Component({
    selector: 'example',
    template: `
            <input #myInput type="text">
            <button (click)="onClick(myInput.value)">Show Input Value</button>
        `
})
export class Example{
    onClick(value){
        console.log(value);
    }
}

src/app.ts

import {Component} from 'angular2/core';
import {Example} from './components/example.component';

@Component({
    selector: 'app',
    directives:[Example],
    template: `
        <example></example>
    `
})
export class App{
}

What's That Syntax?

(click)

The () around and event name in a template indicate an Angular 2 event handler.

The #

The # marks an element to be looked up by other elements. Any value off of the element is available. You can even invoke methods on the element! Look how the following example focuses on the input when you click the button:

<input #example>
<button (click)="example.focus()">Focus the Example Input</button>
